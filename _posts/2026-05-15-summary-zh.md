## [新 Nginx 漏洞 'Rift' 可能导致远程代码执行](https://github.com/DepthFirstDisclosures/Nginx-Rift) ⭐️ 8.0/10

一个名为 'Rift'（CVE-2026-42945）的新 Nginx 漏洞已被披露，在涉及 rewrite 和 set 指令的特定条件下可能导致远程代码执行，而 ASLR 可提供缓解措施。 该漏洞影响全球最广泛使用的 Web 服务器之一 Nginx，可能允许未经身份验证的攻击者实现远程代码执行或拒绝服务，对数百万网站构成重大安全风险。 该漏洞利用需要 rewrite 指令的替换字符串中包含问号，随后在同一作用域内有一个引用未命名 PCRE 捕获组（如 $1）的 set 指令。已发布的 PoC 假设 ASLR 已禁用，但披露方声称存在可靠的 ASLR 绕过方法。

hackernews · hetsaraiya · May 14, 17:17 · [社区讨论](https://news.ycombinator.com/item?id=48138268)

**背景**: Nginx 是一款高性能 Web 服务器和反向代理，被超过 30% 的网站使用。rewrite 模块处理 URL 重写规则，当未命名 PCRE 捕获组与替换字符串中的问号结合时，处理不当会导致堆缓冲区溢出。ASLR（地址空间布局随机化）是一种通过随机化内存地址来增加利用难度的安全技术。
